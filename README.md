# Salon One — 経営ダッシュボード（プロトタイプ）

複数ブランド・多店舗のサロン経営（ヘア／ネイル／アイラッシュ／リラク／エステ／整体）を統合管理する、**経営者向けダッシュボード**のプロトタイプです。
既存の現場向けシステム「**Salon One**」からデータを取得し、経営判断に必要な数値・資金・財務を一画面に集約することを想定しています。

> **位置づけ**：本リポジトリは「機能・デザイン・UX の参照実装（リファレンス）」です。データ連携方式は後決めのため、現在は**決定論的なモックデータ**で動作します。Salon One 開発チームへの共有を前提に、**連携の差し込み口を一箇所に集約**しています（後述）。

> 画面プレビューはチャットに添付したスクリーンショットを参照してください（ライト／ダーク両対応）。

## クイックスタート

```bash

```

Node.js は **20.x** を推奨（`.nvmrc` / `package.json` の `engines` で固定）。

## Vercel へのデプロイ

本構成は Vercel に**ゼロコンフィグ**でデプロイできます（SSR・APIルートにそのまま対応）。

1. Vercel で「**Add New… → Project**」から GitHub リポジトリ `vie324/salon_one_dashboard` をインポート。
2. Framework は **Next.js** が自動検出されます（Build: `next build` / Install: `npm install`）。Root Directory はリポジトリ直下のまま。
3. 環境変数は**現状不要**（モックデータ動作）。Salon One 連携時に、例として以下を Vercel の Environment Variables に追加し、`src/lib/data` から参照します:
   - `SALONONE_API_BASE_URL`（例: `https://api.salonone.net/v1`）
   - `SALONONE_API_TOKEN`
4. **Deploy** を実行 → 本番URLが発行されます。以後 `main` への push で自動デプロイ、PRごとに**プレビューデプロイ**が作成されます。
5. Node ランタイムは **20.x** に固定済み。

> 補足: リポジトリに残る GitHub Pages の `pages-build-deployment` ワークフローは Vercel では不要です（無効化可）。Pages は静的ホスティングのため本フルスタック構成は動作しません。

## 画面一覧（機能）

| ルート | 画面 | 内容 |
|---|---|---|
| `/` | **ダッシュボード** | 全社KPI（売上/営業利益/客数/客単価/リピート/稼働率/キャンセル率/新規）、売上推移（当月は日次＋着地見込）、**要対応アラート（異常自動検知）**、ブランド別構成、決済手段別、本日の状況、店舗ランキング |
| `/sales` | **売上・実績分析** | ブランド／メニュー／スタッフ別、新規/リピート推移、**予約ヒートマップ（曜日×時間帯）**、指名率・再来率 |
| `/customers` | **顧客分析** | アクティブ顧客・LTV・解約率・**RFMセグメント**、サブスク会員/MRR、**前受金（役務）残高** |
| `/marketing` | **マーケティング** | 媒体別（ホットペッパー等）の広告費・新規獲得・**CPA・ROAS**、集客推移 |
| `/cashflow` | **資金繰り・入金管理** | 現金の動き、**各種決済の入金スケジュール（入金済/予定/遅延）**、決済代行別の手数料、前受金残高、サブスクMRR |
| `/reconciliation` | **入金・突合（消込）** | 売上記録と実入金の**差異検出**、確認中/未解決のステータス管理、手段別突合 |
| `/financials` | **財務・PL** | 月次**損益計算書**（構成比・前年比）、**損益分岐点分析**、費用構成、ブランド/店舗別損益 |
| `/stores` | **店舗管理** | 多店舗・多ブランドの横並び比較（売上/利益率/成長率/**人時生産性**）、店舗詳細 |
| `/reports` | **レポート・資料作成** | **月次経営レポートの自動生成**（PDF出力）。役員会・税理士提出資料テンプレート |
| `/settings` | **設定・Salon One 連携** | 連携方式の選択肢、**データ連携マッピング表**、権限（ロール） |

各ページ右上の「PDF出力」はブラウザ印刷を利用し、UI（サイドバー等）を除いた資料として出力できます（`@media print` 対応済）。

## アーキテクチャと「連携の差し込み口」

データ取得は **`src/lib/data`（データアクセス層）** に集約しています。**ここが Salon One 連携の唯一の差し込み口**です。

```
画面（Server Component） ─┐
                          ├─→ src/lib/data の selector（getOverview など）─→ 現状: モック / 将来: Salon One API
/api/[resource]（HTTP）  ─┘
```

- 画面（`src/app/**`）と API（`src/app/api/[resource]/route.ts`）は、**同じ selector** を呼びます。
- 各 selector の戻り値の「形（型）」を保ったまま中身を Salon One API 呼び出しに差し替えれば、**全画面がそのまま動作**します。
- モックの生成元は **`src/lib/data/generate.ts` だけ**です（決定論的乱数で安定。SSRとクライアントで不整合が起きません）。

### データ連携マッピング（ダッシュボード項目 ↔ Salon One）

| ダッシュボード項目 | Salon One 取得元 | 差し替え対象 selector / endpoint |
|---|---|---|
| 売上・客数・予約・キャンセル | 予約管理 / 成績管理 | `getOverview`, `getSales` / `/api/overview`, `/api/sales` |
| 顧客・LTV・リピート・RFM | 顧客管理 / カルテ管理 | `getCustomers` / `/api/customers` |
| メニュー別売上 | カルテ / メニュー | `getSales` / `/api/sales` |
| 決済・入金・手数料 | 決済 / キャンセル料自動請求 | `getCashflow` / `/api/cashflow` |
| 売上 ↔ 入金 突合 | 決済 / 各代行入金明細 | `getReconciliation` / `/api/reconciliation` |
| PL（原価・人件費・家賃 等） | 会計連携 / 経費 | `getFinancials` / `/api/financials` |
| サブスク・前受金（役務） | サブスク管理 / 役務管理 | `getCashflow`, `getCustomers` |
| 集客チャネル | 媒体連携（ホットペッパー等）/ LINE | `getMarketing` / `/api/marketing` |

### 連携方式の選択肢（`/settings` に記載）

- **REST API（推奨）**：Salon One の読み取りAPIを各 selector から呼ぶ
- **Webhook**：予約確定・決済・キャンセル等のイベントで集計を更新
- **バッチ/CSV**：日次・月次のエクスポート取込
- **共有DB（リードレプリカ）**：同一サーバー前提なら参照用レプリカから直接集計

## デザイン / UX のポイント

- **上質・モダン**：ディープインディゴ基調＋金融セマンティクス（増=緑/減=赤）、余白広め、ソフトシャドウ。
- **ライト/ダークモード**：トップバーのトグル。初回ロード時のちらつき防止スクリプト入り。
- **グローバルフィルタ**：期間（当月/先月/直近3・6・12ヶ月/今期）・ブランド・店舗・比較基準（前年同期/前期間）。URL クエリで保持し、ページ遷移でも維持。
- **数値の整列**：財務テーブルは等幅数字（tabular-nums）で桁を揃え可読性を確保。
- **レスポンシブ**：PCはサイドバー（折りたたみ可）、スマホはドロワー＋フィルタ2段表示。
- **異常検知**：売上急減・営業赤字店舗・突合未解決・入金遅延などを自動でアラート化。

## 追加すべき／追加できたら良い機能（ロードマップ提案）

依頼に沿って、経営ダッシュボードとして有効な機能を提案します（一部は本プロトタイプに実装済み）。

- **予実管理（目標 vs 実績）**：月次・店舗別の予算設定と進捗・着地見込。
- **資金繰り予測**：入金スケジュール＋固定費から将来キャッシュ残高を予測。
- **会計ソフト連携**：freee / マネーフォワード等へPL・仕訳エクスポート、**インボイス制度（適格請求書）/ 電子帳簿保存法**対応の証憑管理。
- **役務（前受金）管理の強化**：コース消化スケジュール、失効・返金見込、負債残高の月次推移（実装済の残高管理を拡張）。
- **キャンセル料自動請求の経営可視化**：請求/回収状況、機会損失額。
- **人件費・人時生産性の最適化**：シフト×売上の生産性、適正人員シミュレーション。
- **LTV/コホート分析**：獲得月別の継続・売上推移、媒体別LTV。
- **店舗ベンチマーク/スコアカード**：同業態間の偏差、出店/撤退の意思決定支援。
- **アラートの通知連携**：LINE / Slack / メールへ閾値超過を自動通知。
- **権限・監査ログ**：役員/エリア長/経理/税理士のロール別閲覧範囲（設計を `/settings` に提示）。
- **多通貨・複数法人**：ホールディングス連結、法人横断の集計。
